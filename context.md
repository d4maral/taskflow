Objetivo :

Aplicar os conhecimentos adquiridos ao longo do semestre na disciplina de Teste de Software por meio da realização de testes automatizados em uma aplicação web simples (como uma landing page). O trabalho deverá unir práticas de testes de caixa preta e caixa branca, incluindo o uso de tabelas de decisão como técnica de modelagem.

Descrição da Atividade :

Cada grupo deverá escolher ou desenvolver uma página web funcional — como uma landing page, um formulário ou uma miniaplicação com pelo menos dois fluxos de interação (ex: cadastro, login, envio de mensagem). Com base na aplicação escolhida, o grupo deverá:

1. Planejar os testes, identificando os principais fluxos a serem validados. (Usar uma ferramenta de gerenciamento de Projetos e apresentar)

2. Modelar os casos de teste, utilizando tabelas de decisão para pelo menos dois cenários distintos.

3. Automatizar os testes de interface (testes de caixa preta) com Selenium WebDriver.

4. Desenvolver testes de unidade e integração (testes de caixa branca), utilizando Pytest, quando aplicável.

5. Executar e validar todos os testes, garantindo sua correta implementação.

6. Apresentar o trabalho para a turma, explicando as decisões tomadas, os testes realizados e demonstrando sua execução.

Tecnologias obrigatórias

- Selenium WebDriver (preferencialmente com Python)

- Pytest

- HTML/CSS/JS simples ou frameworks front-end (opcional)

Entrega

- Repositório com o código-fonte da aplicação testada e dos testes automatizados

- PDF contendo:

  - Descrição da aplicação

  - Modelagem dos testes com tabelas de decisão

  - Descrição dos testes implementados

  - Evidências (prints, logs, etc.) da execução dos testes

- Apresentação oral em sala (tempo: até 15 minutos por grupo)

Critérios de Avaliação – Total: 40 pontos

- Modelagem com tabelas de decisão

- Testes automatizados com Selenium

- Testes com Pytest

- Clareza e organização do código e documentação

- Apresentação e demonstração em sala
